## 已完成

- 已把 `SCSI Features` 这一套并行兼容闸口从当前实现中删除，正式收束为“运行时兼容性只看 `VersionBe` 相等”。
- 共享协议头 [windows/shared/yumedisk_proto.h](../../windows/shared/yumedisk_proto.h) 已删除：
  - `YUMEDISK_FEATURE_FLAGS`
  - `YUMEDISK_SCSI_INFO.Features`
- `YumeDiskSCSI` 的 [control/control.c](../../windows/YumeDiskSCSI/YumeDiskSCSI/control/control.c) 已不再填充 `Features` 字段；`QueryScsiInfo` 只返回：
  - `VersionBe`
  - `AdapterSignature`
- 共享协议头进一步继续收束，已删除：
  - `YUMEDISK_KMDF_INFO.ServiceName`
  - `YUMEDISK_SCSI_INFO.ServiceName`
  - `YUMEDISK_KMDF_INFO.Reserved`
  - `YUMEDISK_SCSI_INFO.MaxTargets`
  - `YUMEDISK_SCSI_INFO.Reserved`
- `AppKernel` 的 [ak_session.c](../../windows/AppKernel/src/session/ak_session.c) 已删除：
  - `SCSI app-owned queue` 特性位检查
  - `AkOpen ready` 日志中的 `features=...` 输出
- 正式文档已经同步到当前唯一口径：
  - [windows/AppKernel/AppKernel设计文档.md](../../windows/AppKernel/AppKernel设计文档.md)
  - [windows/AppKernel/AppKernel-SDK文档.md](../../windows/AppKernel/AppKernel-SDK文档.md)
  - [docs/技术文档.md](../技术文档.md)
  - [docs/功能文档.md](../功能文档.md)
  - [docs/项目概述.md](../项目概述.md)
- 同时回看了现有兼容闸口：
  - `AppKernel` 侧当前已只保留 `KMDF.VersionBe == AppKernel.VersionBe` 与 `SCSI.VersionBe == AppKernel.VersionBe` 两个运行时兼容判断。
  - `KMDF` 侧仍保留 `QueryScsiInfo` 的 `AdapterSignature + VersionBe` 探测。
  - 其中 `VersionBe` 属于兼容判断；`AdapterSignature` 只用于确认“当前打开到的是不是我们的 miniport”，不是兼容分叉。
  - 当前没有发现额外的 `Features`、`ProtocolVersion`、双轨协议或 capability gate 继续参与运行时兼容判定。
  - `QueryKmdfInfo / QueryScsiInfo` 现在也只保留真实消费字段，不再挂无业务读取的占位信息。
- 版本规则文档也已与当前真实口径对齐：
  - 对外发行后，协议/语义不兼容变化必须升级版本。
  - 首次正式发行前继续固定使用 `0.1.0.0`，不为未发行阶段的协议收束单独滚动版本号。
