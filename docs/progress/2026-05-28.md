# 2026-05-28

## 已完成

### 1. tauri-client 文件盘尾块处理已改为逻辑补齐，不再丢弃不足盘块的数据

- `windows/tauri-client/src-tauri/src/backend/raw_file_media.rs` 已把 RAW 文件容量模型从“向下截断到盘块边界”改为“按盘块大小向上补齐为逻辑容量”。
- 文件实际大小不足最后一个盘块时：
  - 真实文件内已有字节继续参与磁盘数据读取。
  - 超过真实 EOF 但仍落在逻辑盘容量内的部分，读取固定返回 `0`。
  - 写入跨过真实 EOF 时，只写入真实文件覆盖范围内的前缀；超出真实 EOF 的尾部字节直接丢弃，不写回文件。
- `0` 字节 RAW 文件仍保持无效，不允许作为文件盘打开。

### 2. 文件盘对齐粒度已收口到 BackendRust 的 sector_size，不再写死 512

- `windows/tauri-client/src-tauri/src/backend/config_service.rs` 已新增统一入口，当前从 `backend_rust::DiskConfig::default().sector_size` 获取盘块大小。
- `windows/tauri-client/src-tauri/src/backend/persistence_service.rs` 中 RAW 文件探测与打开已改为显式使用该 sector size。
- `windows/tauri-client/src-tauri/src/backend/disk_service.rs` 与 `windows/tauri-client/src-tauri/src/network/runtime_flow.rs` 中磁盘挂载配置已显式使用同一 sector size，避免探测容量与挂载配置不一致。

### 3. file media 读写链路边界已收束

- 当前修改只落在 `tauri-client` 的 file media 实现层，没有改 `BackendRust` 的 staging / overlay / commit 行为。

  - 逻辑补齐部分读恒为 `0`。
  - 逻辑补齐部分不持久化。
