## 已完成

### 1. tauri-client 文件盘尾块处理已改为逻辑补齐，不再丢弃不足盘块的数据

- `windows/tauri-client/src-tauri/src/backend/raw_file_media.rs` 已把 RAW 文件容量模型从“向下截断到盘块边界”改为“按盘块大小向上补齐为逻辑容量”。
- 文件实际大小不足最后一个盘块时：
  - 真实文件内已有字节继续参与磁盘数据读取。
  - 超过真实 EOF 但仍落在逻辑盘容量内的部分，读取固定返回 `0`。
  - 写入跨过真实 EOF 时，只写入真实文件覆盖范围内的前缀；超出真实 EOF 的尾部字节直接丢弃，不写回文件。
- `0` 字节 RAW 文件仍保持无效，不允许作为文件盘打开。

- `windows/tauri-client/src-tauri/src/backend/persistence_service.rs` 中 RAW 文件探测与打开已改为显式使用该 sector size。
- `windows/tauri-client/src-tauri/src/backend/disk_service.rs` 与 `windows/tauri-client/src-tauri/src/network/runtime_flow.rs` 中磁盘挂载配置已显式使用同一 sector size，避免探测容量与挂载配置不一致。

### 3. file media 读写链路边界已收束

### 8. Unit Attention / DataChanged Phase D 已补齐 BackendRust 单盘下行接口

- `windows/BackendRust/src/appkernel.rs` 已同步 FFI 声明 `AkNotifyDiskDataChanged(...)`，使 Rust 宿主可以直接调用当前正式 AppKernel 下行 API。
- `windows/BackendRust/src/runtime.rs` 已为宿主补上正式入口 `notify_managed_disk_data_changed(target_id, out_error_text)`：
  - 固定只接单盘 `target_id`
  - 先校验 session 已打开
  - 再校验 target 仍存在于当前 `BackendRust` 管理盘集合内
  - 再校验目标盘已有有效 `AK_DISK*` 句柄
  - 再查询当前生命周期，固定只允许 `running` 盘下发
  - 最后同步调用 `AkNotifyDiskDataChanged(...)`
- 当前失败文本已收成最小集合：
  - `session-not-open`
  - `target-not-found`
  - `disk-handle-not-ready`
  - `disk-not-running(<lifecycle>)`
  - `query-disk-state-failed(0xXXXXXXXX)`
  - `notify-data-changed-failed(0xXXXXXXXX)`
- 当前 `BackendRust` 继续明确不做：
  - `smid` 注册表
  - sibling 查找
  - 共享组状态缓存
  - `WriteFinalCommitted -> sibling notify` 隐式 fanout
  - `NotifyDataChanged` 专用后台线程、队列或 retry 状态机
- `AkEventWriteFinalCommitted` 事件线程语义保持不变，仍只负责当前盘 staged write commit，不顺手触发任何其他盘通知。
