### 1. server 侧继续收束 gateway / storer whole 的公共边界

- 将 `gateway` 与 `storer whole` 重复的 `client listener` 外壳抽成公共能力 `ServeClientListener(...)`。
- 公共层只负责：
  - `net.Listen` 后的 `Accept` 循环
  - `ctx.Done()` 时关闭 listener
  - temporary accept error 重试

- `gateway` 和 `storer whole` 继续各自保留自己的连接处理逻辑，没有把数据面、注册面或会话面揉进公共层。
- `gateway` 继续使用现有 `ServeAcceptedClientConnection(...)` 承接 client 连接处理。
- `storer whole` 继续通过内嵌 gateway 复用同一套 client 连接处理能力。
