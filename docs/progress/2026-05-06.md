# 2026-05-06

- 调整 write ACK 协议，移除 `POST_WRITE_SLOT` 输入里的 piggyback ACK 语义。
  - `windows/shared/yumedisk_proto.h` 中的 `YUMEDISK_SUBMIT_SLOT` 已收缩为纯 slot 描述符，不再混入 ACK trailer。
  - `windows/YumeDiskKMDF/YumeDiskKMDF/control/ioctl.c` 已改为强制 `POST_WRITE_SLOT` 输入 payload 为空；独立 `WRITE_ACK_BATCH` 保留为唯一 ACK 入口。
  - `windows/YumeDiskSCSI/YumeDiskSCSI/control/control.c` / `queue/queue.c` 已去掉 write slot 上的 ACK trailer 校验和消费逻辑。
- `WRITE_ACK_BATCH` 增加逐项失败返回。
  - 协议新增 `YUMEDISK_WRITE_ACK_BATCH_RESULT` 与 `YUMEDISK_WRITE_ACK_FAILURE`。
  - SCSI 处理 batch 时，不再因为单个 range 失败直接让整条控制命令失败；而是把失败的 `RangeIndex + Status` 回给 App。
  - 当前找不到 `eventId` 的 ACK range 会返回 `STATUS_NOT_FOUND`，便于后续把 cancelled/stale ACK 与正常落账区分开。
- `windows/tests/RWTestApp/main.cpp` 已切到“仅独立 flush ACK”模型：
  - write worker 不再从 `POST_WRITE_SLOT` 偷带 ACK。
  - `WRITE_ACK_BATCH` 返回中的逐项失败会被打印并计数，便于观察取消模型下的 stale/cancelled ACK。
- 本轮验证：
  - `cmake --build windows/tests/RWTestApp/cmake-build-debug --config Debug` 通过。
  - `YumeDiskSCSI`、`YumeDiskKMDF` 源码编译和链接通过；驱动项目最终失败点仍只有环境侧 `InfVerif.dll` 缺失。
