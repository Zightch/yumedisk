### 2. server 内部 session lifecycle 已补齐 closing/drain

- `server/internal/session/` 已补上 `live -> closing -> drained` 生命周期。
- `Close` / `CloseConnection` 到达时：
  - 立即把 session 置为 `closing`
  - 新的 I/O 直接拒绝
  - 已在途 I/O 允许自然完成
  - 只有在途计数清零后，才真正 drain 并释放 writer 独占位
- 这次收口后，close 路径不再打穿单 writer 约束，也不再依赖旧 close ack 作为同步屏障。

## 当前边界

- 正式协议当前只保留 `SessionCloseNotice`，不再存在独立 `Close` request/response 语义。
- 当前最小闭环实际实现的主动 close 方向仍固定为：
  - `client -> gateway`
  - `gateway -> client`
  - `gateway -> storer`
- `storer -> gateway` 主动 close 仍保留协议留白，但当前项目实现未启用。
- server 内部已经按 `closing/drain` 保护 writer 独占位；协议对外仍不承诺 graceful close。
- `rust-cli` 当前故障策略保持不变：
  - 收到 `SessionCloseNotice` 或 connection 死亡后，直接把对应 network disk 标记清理并卸载
  - 不做自动重连或自动重新挂载
