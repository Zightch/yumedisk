- `windows/tests/MyApp5` 已移除 `wait` 命令和旧 `WaitEvent` 调试路径。
- 完成“Rebuild the protocol entrypoints on top of the skeleton”子步骤。
- `windows/shared/yumedisk_proto.h` 已切换到新协议命令空间，加入 `PostReadSlot / PostWriteSlot / ReadAck / WriteAckBatch / SubmitSlot / CancelSlot` 以及对应 payload 结构，不再保留旧 `WaitEvent / ReadReply / WriteAck` 协议体。
- `windows/YumeDiskKMDF/YumeDiskKMDF/control/ioctl.c` 已改成两类入口：
  - 控制消息继续走 `QueryInfo / CreateDisk / RemoveDisk / RemoveAllDisks` 透传。
  - 新数据面入口已切到 `PostReadSlot / PostWriteSlot / ReadAck / WriteAckBatch / CancelSlot` 语义，并在 KMDF 侧把 slot / ack 转成 `SubmitSlot / ReadAck / WriteAckBatch / CancelSlot` 小描述符转发给 SCSI。
- `windows/YumeDiskSCSI/YumeDiskSCSI/control/control.c` 已识别并校验新协议 payload；当前阶段对 `SubmitSlot / ReadAck / WriteAckBatch / CancelSlot` 在校验通过后统一返回 `STATUS_NOT_SUPPORTED`，明确下一步应由真实队列实现接手。
- `windows/tests/RWTestApp/main.cpp` 与 `windows/tests/MyApp5/main.cpp` 已改成新的 `DeviceIoControl(input=message, output=buffer)` 收包方式，匹配新的 KMDF 边界解释。
- 完成“Rework KMDF session handling around file-bound lifecycle and locked timeout state”子步骤。
- `windows/YumeDiskKMDF/YumeDiskKMDF` 的 session 生命周期已从 device-global 下沉到 file-bound：
  - `CTRL_FILE_CONTEXT` 现在持有 `SessionLock / WatchdogTimer / MiniportHandle / SessionId / LastHeartbeatTick / State`。
  - `CTRL_DEVICE_CONTEXT` 只保留独占打开控制，不再缓存数据面默认 miniport handle。
- `ControlSessionTryOpen` 现在在 App 打开 file object 时立即建立 session、打开 miniport handle、启动 watchdog；`ControlSessionCleanup` 在 file cleanup 时固定执行 `RemoveAllDisks | SessionClose -> close miniport handle -> release open slot`。
- heartbeat 机制已收敛为仅在 KMDF 内维护：
  - `Heartbeat` IOCTL 只更新 KMDF session 的 `LastHeartbeatTick`。
  - `YumeDiskSCSI` 不再承担 heartbeat 机制。
- watchdog 超时后，KMDF 会把该 file session 切到 locked 状态，发送 `RemoveAllDisks | SessionClose` 清盘并取消后端，再关闭自己持有的 miniport handle；在 file object 真正关闭前，该 session 的后续 IOCTL 直接失败。
- `transport` 层已改成基于 file session 工作：
  - `ControlOpenMiniportHandle(...)` 显式打开 file-bound miniport handle。
  - `ControlProxyCommand(...)`、`ControlCloseMiniportHandle(...)`、`ControlSendSessionCleanup(...)` 都改为围绕 `CTRL_FILE_CONTEXT` 工作。
- `control/ioctl.c` 已改为从 `WdfRequestGetFileObject(Request)` 定位 session；除 heartbeat 外，其余命令都先获取 file session 再透传，避免 handle/state 与 watchdog cleanup 竞态。
- `windows/tests/RWTestApp/main.cpp` 已增加自动 heartbeat 后台线程；`windows/tests/MyApp5/main.cpp` 已增加手工 `heartbeat` 命令，便于 session 生命周期调试。

## 已收束边界

- 当前仓库已经没有旧 `WaitEvent` 数据面主路径。
- 当前骨架阶段只保留控制面闭环，读写压测能力暂时故意断开。
- 下一步必须从新协议入口开始重建，不能回退到旧 event inline data 模型。
- 当前新协议入口已经固化，但 slot 生命周期、heartbeat watchdog、真实 SCSI 队列与 App backend 仍未实现；新数据面命令现在是“结构已定、入口已通、行为未落”的停止线。
- heartbeat/watchdog 的唯一真实来源已经收敛到 KMDF file session；SCSI 不再承担 App 存活探测职责。
- miniport handle 生命周期已经与 App file object 绑定，不再允许回退到 device-global lazy cached handle 模型。
- session 一旦 watchdog 超时进入 locked 状态，在 file object 关闭前不允许通过新的 heartbeat 或新的 slot 请求复活。
