## 已收束边界

- 当前不回退 `App per-disk bounded workers` 数据面模型。
- 当前不回退 `SCSI per-target queue`，也不把 `SCSI` 队列锁改成 wait lock。
- 当前不改协议边界，仍只保留：
  - `POST_READ_SLOT`
  - `POST_WRITE_SLOT`
  - `READ_ACK`
  - `WRITE_ACK_BATCH`
  - `CANCEL_SLOT`
- 当前不改取消模型：
  - 系统侧取消只追到 `SCSI`。
  - App 可以回晚到 ACK。
  - stale / cancelled / not found 仍由 `SCSI` 判定。
- 当前 `KMDF` 热路径已经删掉的旧结构，不应再恢复：
  - per-slot `IoAllocateWorkItem / IoQueueWorkItem / IoFreeWorkItem`
  - per-slot `IoAllocateIrp / IoFreeIrp`
  - `POST_*_SLOT` 每次进入 `SessionLock`
  - submit queue 每 slot 单次抢锁后立即 `IoCallDriver`
- 当前剩余验证缺口已经明确收缩到运行面，而不是设计面：
  - 需要在 VM 中按 `ct -> disk visible -> Q1/Q2/Q8/Q32 -> Ctrl+C while active -> rm all -> app exit` 验证吞吐、kernel CPU、cleanup/watchdog 收尾和 late ACK 语义是否仍稳定。
- `AppKernel` 当前只完成了 Step 1 骨架，不应把这些占位实现误当作已可承接真实数据面的会话层：
  - 还没有真正打开 `KMDF control device`。
  - 还没有 heartbeat 线程。
  - 还没有异步 slot transport。
  - 还没有 per-disk runtime。
