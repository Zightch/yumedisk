# 2026-05-31

## 已完成

### 1. 网络 `data changed` 与 `backend_id` 正式口径已补齐

- `docs/network` 已正式定义 `SessionDataChangedNotice`，并明确它与 `SessionCloseNotice` 分离：
  - `data changed` 只表示底层内容已变更
  - 不表示会话失效
  - 不触发 network session 关闭语义
- `docs/network` 已正式定义 `SessionDescribe` metadata 增加 `backend_id`。
- metadata 真源已在正式文档中收口为 `storer`：
  - `gateway` 不再作为 metadata 真源
  - `gateway` 不再维护 metadata snapshot 的正式口径

### 2. server 已完成 `rw committed -> ro notice` 最小闭环

- server 协议与编解码已补 `SessionDataChangedNotice`。
- `storer` 已在 `rw` 写入真正 committed 后：
  - 枚举同 backend 下的 live `ro session`
  - 向上游逐个发送 `SessionDataChangedNotice`
- `gateway` 已改为只做 session 映射与转发：
  - 按 `storer_session_id -> gateway_session_id` 转发 notice
  - 目标 session 失效时幂等忽略
- `whole` 角色已走同一条内嵌网关能力路径，不再分叉出另一套 metadata/data changed 真源逻辑。

### 3. `backend_id` 已落到 storer metadata 真源

- `backend_id` 由 `storer`/`whole` 在程序启动时临时生成。
- 同一个本地 backend 的 `rw` 和 `ro` 导出共用同一个 `backend_id`。
- `backend_id` 不持久化，不要求跨重启稳定。
- `backend_id` 只通过 `SessionDescribe` 暴露：
  - 不进入 `AuthStart/AuthFinish`
  - 不进入 `SessionOpen`
  - 不进入 gateway route/auth/session 真状态

### 4. rust-cli 已接入 `data changed` 与 `remote_backend_id`

- `windows/network-core` 已支持：
  - `SessionDataChangedNotice` 编解码与接收
  - `SessionDescribe` metadata 中的 `backend_id`
- `rust-cli` 当前最小闭环行为已收为：
  - 收到 `SessionDataChangedNotice` 后，仅触发本地 `notify_managed_disk_data_changed(...)`
  - 不因 `data changed` 失效磁盘或清理 session
- `rust-cli` 已补重复盘拒绝：
  - `(server_addr, remote_disk_id)` 相同拒绝
  - `(server_addr, remote_backend_id)` 相同拒绝

### 5. Windows `BackendRust` 链接面已补齐

- `windows/BackendRust/build.rs` 已显式链接 `appkernel`，确保 `AkNotifyDiskDataChanged` 等符号能稳定解析。
