### 3. `backend_id` 已落到 storer metadata 真源

- `backend_id` 由 `storer`/`whole` 在程序启动时临时生成。
- 同一个本地 backend 的 `rw` 和 `ro` 导出共用同一个 `backend_id`。
- `backend_id` 不持久化，不要求跨重启稳定。
- `backend_id` 只通过 `SessionDescribe` 暴露：
  - 不进入 `AuthStart/AuthFinish`
  - 不进入 `SessionOpen`
  - 不进入 gateway route/auth/session 真状态

  - `vm_win11` 使用 `rust-cli` 挂载 `ro` 导出
- 两台虚拟机最终都拿到同一块 `RAW` 网络盘：
  - Win10: `PhysicalDrive1`, `PartitionStyle=RAW`, `IsReadOnly=False`
  - Win11: `PhysicalDrive1`, `PartitionStyle=RAW`, `IsReadOnly=True`
- 为避免卷/文件系统缓存语义干扰，本轮正式黑盒证据完全走原始盘路径：
  - `rw` 侧直接对 `\\\\.\\PhysicalDrive1` 偏移 `4096` 写入 `512B` marker
  - `ro` 侧先确认同偏移原始字节为全 `00`

  - 第一次返回 `scsi_status=0x02(CHECK_CONDITION)`
  - 第二次恢复 `scsi_status=0x00(GOOD)`
  - 最后在 `ro` 侧同偏移原始读取到刚写入的 marker 字节
- 这条证据表明：
  - 网络 `rw committed -> ro SessionDataChangedNotice -> client notify_managed_disk_data_changed(...)` 已真实打通
  - `ro` 侧原始 `PhysicalDriveN` 路径可以收到这次 `data changed`
  - 本轮结论不依赖文件系统重扫、卷缓存同步或重新挂载

### 11. `tauri-client` Phase 3 四段重扫流水已完成

- `network/runtime_flow` 已从单文件改为目录化结构：
  - `mod.rs` 只保留挂载、拔出、删除准备等入口
  - `rescan.rs` 独立承载网络盘重扫主线
- 网络盘重扫当前已按四段执行流水重建：
  - 先收集全部网络盘任务
  - 再按 `server_addr` 汇总分组
  - 再为每个盘获取 fresh candidate
  - 最后统一裁决并统一提交
- 当前重扫已不再边拿结果边落状态：
  - live session 可复用时直接进入 candidate
  - live session 不可复用时走 `connect -> auth -> open -> describe`
  - fresh describe 不再即时写入 `runtime_store`
  - backend 冲突统一在第四段裁决
- `(server_addr, remote_backend_id)` 冲突当前已按最小闭环正式收口：
  - 同 backend 只有一个盘时正常保留
  - 同 backend 若有一个或多个 `ro`，只保留一个 `ro`
  - 保留者固定为 `local_disk_id` 数值最小的那个 `ro`
  - 其余 `rw/ro` 一律 `invalid`，原因固定为“网络盘后端冲突”
- mounted 网络盘在重扫中的口径已进一步收紧：
  - 只有“复用当前 live session”时才允许保持 mounted
  - 只要本轮拿到的是 fresh reopened session，即使原来已挂载，也必须回到 unmounted
  - 这样可以避免 `NetworkMedia` 继续绑定旧会话
- 冲突失败和旧 live session 替换的清理已统一进入提交阶段：
  - 旧 opened session 统一从 `NetworkClientState` 移除
  - loser/fresh 临时 session 统一 `SessionCloseNotice + cleanup`
  - 再按连接引用情况决定是否做 connection release
- 本轮新增测试覆盖：
  - live session 复用时 unmounted 保持有效
  - live session 复用时 mounted 保持 mounted
  - stale mounted runtime 重新打开 fresh session 后回到 unmounted
  - 同 backend 的 `rw + ro` 只保留 `ro`
  - 同 backend 的多个 `ro` 只保留 `local_disk_id` 最小者
