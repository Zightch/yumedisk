## 已完成

### 1. rust-cli 侧继续收口为单一网络协议模型

- `auth` 继续复用已存在的 `connection`，但一个 `connection` 同时只允许一个认证过程。
- `SessionOpen` 继续复用已认证结果，但一个 `connection` 同时只允许一个打开会话过程。
- 已打开会话允许并发读写，不影响新的会话读取与后续会话挂载。
- `SessionOpen` 的拒绝语义已从模糊 `busy` 收口为明确 `open reject`。
- `NetworkMedia` 的清理策略已收口为“会话关闭时由 client 侧决定是否卸载并清理连接”，只在 session 关闭后再判断 connection 是否还可回收。

### 2. server 侧完成 gateway / storer 职责边界重建

- `gateway` 和 `storer` 已明确为两个独立程序。
- `role` 只保留 `whole` 和 `storer` 两种选项。
- `whole` 现在明确表示“自带网关能力的 storer”，对 client 暴露监听口，但不对其他 storer 暴露 storer 监听口。
- `storer` 角色只负责主动连接独立 `gateway`，走注册和透传，不启动内嵌 `gateway`。
- `SessionOpen` 的 busy 收口已统一为最小闭环下的 `open reject`。
