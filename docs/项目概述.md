# 项目概述

## 1. 项目目标

本项目要实现一个由应用层主导的数据盘型内存虚拟磁盘系统。系统中的虚拟盘由 `MyDisk2` 以 Storport SCSI miniport 的形式向 Windows 暴露，但磁盘内容和磁盘生命周期由应用层 `MyApp5` 主导控制。

为了解决应用层在线状态、设备发现和进程退出感知问题，引入一个伴随的 KMDF 控制驱动 `MyCtl`，基于现有 `MyKWDF3` 演进。

最终目标不是做一个传统内核自包含 ramdisk，而是做一个三段式架构：

- `MyApp`：内存后端、控制平面、协议参与者
- `MyCtl`：会话入口、生命周期守卫、内核代理层
- `MyDisk`：SCSI设备实现、系统I/O入口、盘枚举与热插拔

## 2. 使用范围

该系统只面向以下场景：

- 普通数据盘
- 应用软件直接读写的数据卷
- 测试型、专用型、受控环境中的虚拟存储

明确不支持以下场景：

- 系统盘
- 启动盘
- 分页盘
- 依赖用户态回包才能维持系统稳定的核心卷

## 3. 当前项目映射

当前仓库中不新建独立代码库，直接在现有项目上修改：

- `MyKWDF3` 作为 `MyCtl`
- `MyDisk2` 作为 `MyDisk`
- `MyApp5` 作为 `MyApp`

文档中的逻辑名和仓库中的物理目录名关系如下：

- `MyCtl` = `MyKWDF3`
- `MyDisk` = `MyDisk2`
- `MyApp` = `MyApp5`

## 4. 总体设计原则

### 4.1 App不在线即无盘

这是本项目最重要的运行原则。只要 `MyApp` 没有在线会话，系统中就不应存在任何由 `MyDisk` 暴露的数据盘。

### 4.2 启动即自安装与自修复

`MyApp` 启动后不能假设系统已经处于正确状态，而是必须执行一次完整的环境收敛：

- 检查 `MyCtl` 驱动包是否已安装
- 检查 `MyDisk` 驱动包是否已安装
- 检查 `MyCtl` 设备实例是否存在且唯一
- 检查 `MyDisk` 设备实例是否存在且唯一
- 缺失则安装和创建设备
- 重复则删除多余实例

### 4.3 全链路唯一性

系统必须保证以下对象唯一：

- `MyApp` 活动进程唯一
- `MyCtl` 有效设备实例唯一
- `MyDisk` 有效设备实例唯一
- `MyCtl` 控制会话唯一

### 4.2 会话入口和SCSI入口分离

`MyApp` 不直接操作 `GUID_DEVINTERFACE_STORAGEPORT`。  
`MyCtl` 对外提供稳定的自定义设备接口，负责：

- 感知 `CreateFile`
- 感知进程退出
- 代理自定义协议
- 在 App 崩溃时立即触发删盘

### 4.4 真实系统I/O由MyDisk负责

系统层面对磁盘的 `READ/WRITE/INQUIRY/READ_CAPACITY` 等 SCSI 请求，仍然全部进入 `MyDisk`。  
`MyCtl` 不拦截普通文件系统 I/O，也不做磁盘过滤。

### 4.5 读写都必须事务化

系统读盘和写盘都必须形成事务，由 `MyDisk` 分配事务 ID，通过事件交给 `MyApp`，再由 `MyApp` 进行应答。

## 5. 运行流程概述

### 5.1 启动阶段

1. `MyApp` 先抢占单实例锁
2. `MyApp` 检查 `MyCtl` 与 `MyDisk` 驱动包是否存在，缺失则安装
3. `MyApp` 检查 `MyCtl` 与 `MyDisk` 设备实例是否存在，缺失则创建
4. `MyApp` 检查 `MyCtl` 与 `MyDisk` 是否存在重复实例，存在则清理多余实例
5. `MyCtl` 启动
6. `MyDisk` 启动
7. `MyDisk` 默认不暴露任何盘
8. `MyApp` 枚举 `MyCtl` 自定义 GUID 并打开设备
9. `MyCtl` 建立会话，并在内核中定位 `MyDisk`

### 5.2 创建磁盘

1. `MyApp` 建立心跳
2. `MyApp` 预投放 `WAIT_EVENT` 请求，目标深度为 64
3. `MyApp` 通过 `APP_CMD_CREATE_DISK` 请求建盘
4. `MyCtl` 转发给 `MyDisk`
5. `MyDisk` 将目标 Target 标记为 present
6. `MyDisk` 通知总线变化，系统重新枚举出新磁盘

### 5.3 正常I/O

1. 系统向 `MyDisk` 发起读写
2. `MyDisk` 生成事务并等待 `MyApp`
3. `MyDisk` 通过 `WAIT_EVENT` 把事务发送给 `MyApp`
4. `MyApp` 处理后用 `APP_CMD` 回包
5. `MyDisk` 依据回包完成原始 SCSI 请求

### 5.4 退出与删盘

1. `MyApp` 正常退出或异常退出
2. `MyCtl` 在 `EvtFileCleanup` 感知会话结束
3. `MyCtl` 通知 `MyDisk` 删除全部磁盘
4. `MyDisk` 失败完成所有未决事务
5. `MyDisk` 通知总线变化
6. 系统将这些虚拟盘视作被拔出

## 6. 交付结果

项目完成后，预期具备以下结果：

- `MyApp` 启动后能自动把驱动与设备状态收敛到唯一正确形态
- 应用层可动态创建和删除多个虚拟盘
- 系统可将这些虚拟盘识别为普通数据盘
- 读写路径由 App 主导
- App 退出后虚拟盘立即消失
- 多盘按多 Target 方式暴露，不使用多 LUN 方案
