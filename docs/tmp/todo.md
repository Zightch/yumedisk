# 当前总目标

重建 `windows/tauri-client` 页面与功能最小闭环。

目标收口为：

- 先不上任何自定义 `CSS`，严格遵循 `Element` 法则，只用 `Element Plus` 现成布局、表单、弹层、消息和基础组件。
- 先重建与概念页布局相同的主页骨架，不额外增加概念页之外的入口、字段、按钮和展示块。
- 完成 `BackendRust` 接入、`session` 打开、初始化页切换到主页这条唯一主路径。
- 完成内存盘 / 文件盘创建，但文件盘当前只支持“选择现有文件”，不做“创建文件”分支。
- 完成盘列表配置持久化到家目录 `.yumedisk`。
- 明确“创建盘配置”与“连接到系统”分离：创建后立即在 `tauri` 宿主侧创建并持有 `Media` 实例，同时写配置和更新列表，但不自动连接；只有用户在列表中手动连接时，才把当前已持有的 `Media` 实例移交给 `BackendRust` 发送建盘请求。

# 子步骤

1. 重建 `tauri-client` 顶层两态页面骨架
   - 只保留初始化页和主页两态。
   - 初始化页负责打开 `session` 并等待结果。
   - 主页布局必须与概念页一致，但当前不补主题、动画和样式细节。

2. 重建严格按概念页收口的主页 Element 骨架
   - 顶部只保留 `YumeDisk`、设置入口、会话状态、添加按钮。
   - 主体只保留磁盘列表区域。
   - 列表项只保留概念页已有字段和操作，不新增调试信息。

3. 重建 `BackendRust` 宿主桥接最小入口
   - 在 `src-tauri` 建立 `session`、`disk`、`config` 最小 command。
   - 打通前端初始化调用到 `BackendRust open session`。
   - 保持 `src-tauri` 为薄桥接，不在 Rust 侧堆页面流程逻辑。

4. 重建盘配置真状态与 UI 展示映射
   - 运行中真实盘状态、配置状态、展示 DTO 各自边界明确。
   - 前端只消费宿主返回快照，不复制一份长期运行态真状态。
   - 列表数据以“已保存盘配置 + 当前连接状态”组合展示。

5. 重建内存盘创建流程
   - 从添加按钮进入类型选择，再进入内存盘创建对话框。
   - 当前只支持必要字段：名称、容量、介质选择、是否启动自动连接。
   - `auto` 只是创建策略，不是最终盘类型；保存时必须解析成最终介质或保存为明确的创建策略字段，不能让运行态盘类型语义混乱。

6. 重建文件盘创建流程
   - 从添加按钮进入类型选择，再进入文件盘创建对话框。
   - 当前只支持“选择现有文件”。
   - 当前不做“创建文件”、不做多格式创建、不做历史路径记忆。

7. 重建盘配置持久化
   - 配置固定保存到家目录 `.yumedisk`。
   - 当前至少保存：客户端最小配置、盘列表配置、启动自动连接标记。
   - 配置保存的是“希望恢复什么”，不是“当前已经连接了什么”。

8. 重建手动连接 / 断开主路径
   - 创建盘后默认只出现在列表中，不挂盘、不创建系统可见盘。
   - 添加磁盘后，宿主侧立即创建并持有对应 `Media` 实例。
   - 用户点击连接时，宿主侧把当前持有的 `Media` 实例移交给 `BackendRust`，并发送正式建盘请求。
   - 用户点击断开时，宿主侧发送删盘请求，并重新接回该 `Media` 实例而不销毁，再把列表状态切回未连接。
   - 这样内存盘在当前 App 常驻期间可以断开后重新连接且数据不丢。
   - App 退出后，内存盘 `Media` 实例随进程销毁，内存数据丢失；重新启动只能按配置重建空介质。

9. 重建最小验收路径
   - 启动客户端。
   - 初始化页打开 `session` 成功并切主页。
   - 创建一个内存盘配置并保存到列表。
   - 创建一个文件盘配置并保存到列表。
   - 手动连接其中一个盘并在列表中看到状态变化。
   - 退出并再次启动后，从 `.yumedisk` 恢复盘配置列表。

# 当前轮边界

- 当前只做 `tauri-client` 的最小页面与功能闭环，不做 UI 美化。
- 当前严禁为贴概念稿而补自定义 `CSS`，严禁手写超出 `Element Plus` 必要覆盖范围的样式体系。
- 当前不做主题系统、动画系统、通知视觉微调、响应式精修。
- 当前不做网络盘。
- 当前不做文件盘“创建文件”分支。
- 当前不做 `rawFile` 之外的文件介质格式。
- 当前不做自动连接恢复执行链；只先保存配置与启动标记，恢复执行可以作为下一轮单独目标。
- 当前不把“创建盘配置”和“连接建盘”混成一个动作。
- 当前由 `tauri` 宿主长期持有未连接盘的 `Media` 实例；连接时移交给 `BackendRust`，断开时再接回宿主。
- 当前 `Media` 实例持有只保证 App 常驻期内有效，不承诺跨进程保活；内存盘在 App 退出后数据丢失。
- 当前不引入概念页之外的额外页面、额外统计、额外调试入口、额外设置项。
