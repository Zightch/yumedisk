# 当前总目标

重建 `windows/tauri-client` 的会话状态入口，完成“主页顶部会话状态按钮 + 会话状态对话框 + 失败重试入口”的最小闭环，并与已经完成的启动链、启动重扫和启动期磁盘展示覆盖保持单一职责边界。

## 当前未完成子步骤

1. 重建顶部会话状态按钮
   - 将当前顶部“会话正常 / 正在初始化 / 会话失败”状态块重建为可点击按钮。
   - 保留现有视觉基调，只在必要范围内接管和覆写 `Element Plus` 按钮样式。
   - 初始化态显示旋转指示 + “正在初始化”。
   - 正常态显示绿圆 + “会话正常”。
   - 失败态显示红圆 + “会话失败”。
   - 按钮点击统一打开会话状态对话框，不在按钮本体上叠加额外控制逻辑。

2. 重建会话状态对话框
   - 新增独立会话状态对话框，用于承接会话状态查看与失败重试。
   - 对话框内容沿用当前初始化页的状态骨架，但不恢复独立初始化页面。
   - 初始化态展示旋转指示、标题文案和当前阶段说明。
   - 正常态展示成功状态与当前 `stateText`，不显示重试按钮。
   - 失败态展示失败状态、错误说明，并显示重试按钮。

3. 重建失败重试链路
   - 会话状态对话框中的重试按钮只重试“打开会话 -> 启动重扫 -> 自连接”。
   - 重试不重复恢复配置，不重新构建主页，不重新反序列化磁盘配置。
   - 重试成功后退出失败覆盖，恢复真实磁盘展示态，并再执行一次自连接。
   - 重试失败后继续停留在统一失败覆盖态。

4. 收紧主页边界与交互语义
   - 主页上的启动链、失败覆盖和状态对话框入口继续只保留一套实现。
   - 用户手动重扫继续保留现有按钮入口，不与会话重试混用。
   - 托盘重新显示窗口时，不重复执行会话重试或启动自连接。
   - 不把“会话失败 / 正在初始化”写入 Backend 真实磁盘状态。

5. 文档归档与阶段验收
   - 完成每个子步骤后，立即同步 `docs/progress/2026-05-14.md`。
   - 如果会话按钮、对话框和失败重试入口形成长期规则，同步更新正式 UI / 架构文档。
   - 当前阶段验收只覆盖：
     - 程序启动直接进入主页；
     - 启动期磁盘统一显示无效覆盖；
     - 会话成功后自动重扫并恢复真实未连接状态；
     - 顶部会话状态按钮正确反映初始化 / 正常 / 失败；
     - 失败后点击状态按钮可打开对话框并执行重试。

## 当前轮边界

- 当前轮只收 `windows/tauri-client` 的会话状态按钮、会话状态对话框和失败重试入口。
- 当前轮不回退到独立初始化页，不重建新的双轨启动结构。
- 当前轮不扩展运行期 `session broken` 自愈机制，不新增后台自动重连体系。
- 当前轮不新增 Backend 真实磁盘状态类型，不把展示覆盖态写回真实状态。
- 当前轮继续坚持 `Element Plus` 为基础，只在实现目标视觉和交互语义时做必要接管。
