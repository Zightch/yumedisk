# 当前总目标

重建 `windows/tauri-client` 页面与功能最小闭环。

目标收口为：

- 先不上任何自定义 `CSS`，严格遵循 `Element` 法则，只用 `Element Plus` 现成布局、表单、弹层、消息和基础组件。
- 先重建与概念页布局相同的主页骨架，不额外增加概念页之外的入口、字段、按钮和展示块。
- 完成 `BackendRust` 接入、`session` 打开、初始化页切换到主页这条唯一主路径。
- 完成内存盘 / 文件盘创建，其中文件盘同时覆盖“选择现有文件”和“创建 RAW 文件”两条路径。
- 完成盘列表配置持久化到家目录 `.yumedisk`。
- 明确“创建盘配置”与“连接到系统”分离：创建后立即在 `tauri` 宿主侧创建并持有 `Media` 实例，同时写配置和更新列表，但不自动连接；只有用户在列表中手动连接时，才把当前已持有的 `Media` 实例移交给 `BackendRust` 发送建盘请求。

# 子步骤

1. 重建最小验收路径
   - 启动客户端。
   - 初始化页打开 `session` 成功并切主页。
   - 创建一个内存盘配置并保存到列表。
   - 通过“选择现有文件”创建一个文件盘配置并保存到列表。
   - 通过“创建文件”创建一个新的 `RAW` 文件盘配置并保存到列表。
   - 手动连接其中一个有效盘并在列表中看到状态变化。
   - 手动断开该盘并确认列表状态恢复为未连接。
   - 删除一个盘并确认列表与配置同步移除。
   - 把一个文件盘目标改成无效路径后重启，确认客户端可继续进入主页，且该盘显示为无效且不可连接。
   - 再次启动后，从 `.yumedisk` 恢复剩余盘配置列表。

# 当前轮边界

- 当前只做 `tauri-client` 的最小页面与功能闭环，不做 UI 美化。
- 当前严禁为贴概念稿而补自定义 `CSS`，严禁手写超出 `Element Plus` 必要覆盖范围的样式体系。
- 当前不做主题系统、动画系统、通知视觉微调、响应式精修。
- 当前不做网络盘。
- 当前文件盘“创建文件”分支只支持 `RAW`；`VMDK` / `VHD` / `VHDX` / `VDI` / `QCOW2` 仅保留禁用占位。
- 当前不做自动连接恢复执行链；只先保存配置与启动标记，恢复执行可以作为下一轮单独目标。
- 当前不把“创建盘配置”和“连接建盘”混成一个动作。
- 当前由 `tauri` 宿主长期持有未连接盘的 `Media` 实例；连接时移交给 `BackendRust`，断开时再接回宿主。
- 当前 `Media` 实例持有只保证 App 常驻期内有效，不承诺跨进程保活；内存盘在 App 退出后数据丢失。
- 当前不引入概念页之外的额外页面、额外统计、额外调试入口、额外设置项。
