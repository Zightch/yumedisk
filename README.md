# YumeDisk

面向 Windows 的块设备能力平台。

一块内存可以变成磁盘，一个文件可以变成磁盘，**远端的一块存储空间也可以变成磁盘**。

不需要 U 盘、不需要额外硬盘——应用看到的始终是"本地磁盘"，底层介质怎么换，上层都不用管。

YumeDisk 把磁盘操作从物理介质中解放出来。传统上，磁盘就是 U 盘、机械硬盘、固态硬盘——形态固定、数量有限。现在，任何能提供块存储能力的介质都能成为系统磁盘。介质的选择权交给应用层，而不是被硬件形态绑死。

这不是文件传输工具，而是磁盘语义的统一入口。现有方案多能解决访问、同步或共享问题，却无法让应用像操作本地磁盘一样工作。YumeDisk 选择另一条路径——将复杂性沉入底层，将结果呈现为统一、直接、可扩展的磁盘能力。

## 为什么需要它

现有方案各自解决了一部分问题，但在磁盘语义这一层始终存在边界。

### SMB / NFS

适合文件共享，但核心仍是目录与文件访问语义，而非磁盘语义。

- 依赖固定协议和网络环境
- 更适合共享目录，不适合直接充当系统磁盘
- 跨公网或受限环境下部署成本高

### FTP / 网盘 / 同步盘

强调传输、同步与访问入口，但不强调"像本地磁盘一样工作"。

- 本质是文件交换与同步工具
- 工作流仍需上传、下载或额外适配
- 无法直接承接依赖本地磁盘语义的软件与场景

### iSCSI

能提供块设备语义，但部署与网络模型偏重。

- 配置链路较长
- 对网络与运维环境要求高
- 在轻量部署、共享 IP 或灵活宿主控制场景下不够自然

YumeDisk 要补上的正是这些方案之间的空白——既不退回纯文件传输，也不继承传统块存储的沉重部署，而是以更可控的方式将存储能力直接交付为磁盘体验。

## 项目定位

YumeDisk 不是文件同步器，也不是传统网络共享工具的替代外壳。它是一个磁盘能力平台：

- **统一体系**——存储、挂载、运行时管理与扩展能力置于同一框架
- **渐进接轨**——本地与远端、临时与持久、实验与正式部署逐步融合，而非彼此割裂
- **能力生长**——以磁盘语义为锚点，持续承接更多介质与场景

## 为什么聚焦 Windows

| | Linux | Windows |
|---|---|---|
| 块设备接入 | "一切皆文件"，回环、用户态接入路径直接 | 磁盘语义更严格，须经过驱动层与设备模型 |
| 挂载链路 | 系统原生支持，无需额外驱动工程 | 必须走正式挂载链路，无法停留在应用层 |
| 生态特点 | 开发者导向，能力组织灵活 | 生态覆盖广，兼容对象丰富，但准入壁垒高 |

YumeDisk 的意义在于正面打通 Windows 的磁盘语义壁垒，而不是绕开它。

壁垒打通之后，呈现给上层的不仅是驱动或工具，而是**系统磁盘操作向应用层的稳定暴露**。这意味着介质的选择权真正回到了应用手中——磁盘不再局限于 U 盘、机械硬盘、固态硬盘等物理形态，任何能够提供块存储语义的介质都可以被挂载为系统磁盘。内存、文件、远端存储，乃至未来更多非传统介质，都能在同一体系下被应用无缝使用。

应用无需重新发明存储工作流，也无需关心底层介质的差异——围绕统一的磁盘语义，存储不再是零散的外挂组件，而能自然进入现有软件、工具链与工作流。介质变化尽量留在底层，应用层始终获得统一、稳定、可扩展的磁盘操作入口。

## 项目特点

- **磁盘语义优先**——以磁盘语义为中心，而非文件传输语义
- **介质自由化**——打破物理介质限制，内存、文件、网络皆可成为磁盘，不再局限于 U 盘、机械、固态等传统形态
- **统一挂载与管理**——不同介质在同一体系中被承载和调度
- **渐进式演进**——适合逐步扩展，而非一次性押注单一形态

## 当前能力

YumeDisk 已不停留在概念层，而是沿可运行、可管理、可扩展的方向持续收敛。目前已覆盖三类典型介质：

| 介质 | 对应方向 | 说明 |
|---|---|---|
| 内存 | 临时介质 | 易失性块设备，适合缓存与测试 |
| 文件 | 本地持久介质 | 文件承载的虚拟磁盘，即开即用 |
| 网络 | 远端能力入口 | 通过网关接入远端存储，向分布式扩展 |

三者构成了项目继续向外扩展的起点。

## 项目潜力

YumeDisk 的价值不只是"再提供一种存储方式"，更在于为磁盘能力提供了可继续生长的承载面：

- 更丰富的介质类型与远端能力
- 更统一的客户端体验
- 更适合开发、测试、隔离与正式部署的使用方式

它具备成长为平台的可能，而非停留在孤立工具或一次性实验。对 Windows 而言，这一点尤为关键——一旦磁盘语义壁垒打通，后续存储能力就有了进入系统工作流的统一入口。

YumeDisk 欢迎更多开发者参与。你可以：

- **贡献新的介质驱动**——让云存储、数据库，都能变成本地磁盘挂载
- **完善现有能力**——优化性能、增强稳定性、扩展使用场景
- **分享使用案例**——帮助其他人了解 YumeDisk 能解决什么问题
- **提出想法与反馈**——无论是功能建议还是问题报告，都能让项目更好

无论你是想让自己的存储方案"变成磁盘"，还是想体验不一样的磁盘使用方式，这里都有你可以发挥的空间。

## 总结

一句话概括：

> 在 Windows 生态下打通磁盘语义壁垒，将系统磁盘操作交到应用层手中，让介质摆脱物理形态限制，把存储能力组织成可用、可管理、可扩展的磁盘系统体验。
