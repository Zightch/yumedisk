# YumeDisk

面向 Windows 的块设备能力平台。

YumeDisk 致力于将不同来源的存储能力整理为系统可识别、可挂载、可管理的磁盘对象，让应用继续以“本地磁盘”的方式工作。

它关注的不是单纯的文件传输，而是磁盘语义本身。很多现有方案能够解决访问、同步或共享的问题，却无法自然承接系统级磁盘体验。YumeDisk 选择的是另一条路径：将复杂性沉入底层，将结果呈现为统一、直接且可扩展的磁盘能力。

## 为什么需要它

现有方案各自解决了一部分问题，但在磁盘语义这一层始终存在明显边界。

### SMB / NFS

它们适合做文件共享，但核心仍然是目录与文件访问语义，而不是磁盘语义。

- 依赖固定协议和网络环境
- 更适合共享目录，不适合直接扮演系统磁盘
- 在复杂网络、跨公网或受限环境下部署成本更高

### FTP / 各类网盘 / 同步盘

这类方案强调传输、同步与访问入口，但通常并不强调“像本地磁盘一样工作”。

- 更像文件交换和同步工具
- 很多工作流仍然需要上传、下载、同步或额外适配
- 很难直接承接依赖本地磁盘语义的软件和场景

### iSCSI

iSCSI 能提供块设备语义，但其部署模型与网络模型通常更重。

- 配置链路更长
- 对网络与运维环境要求更高
- 在轻量部署、共享 IP 或更灵活的宿主控制场景下不够自然

YumeDisk 希望补上的，正是这些方案之间留下的空白：既不退回到纯文件传输，也不完全继承传统块存储方案的沉重部署方式，而是以更可控的方式，将存储能力直接交付为磁盘体验。

## 项目定位

YumeDisk 不是文件同步器，也不是传统网络共享工具的替代外壳。

它是一个磁盘能力平台。其核心目标，是将存储、挂载、运行时管理与后续扩展能力置于同一条体系之中，使本地与远端、临时与持久、实验性与正式化部署能够逐步接轨，而不是彼此割裂。

## Windows 与 Linux 的差异

在 Linux 生态中，这件事通常更自然一些。Linux 长期强调“一切皆文件”的系统传统，在块设备、挂载、回环与用户态接入等方面，路径往往更加直接，很多能力不必额外引入专门的驱动工程，就可以完成组织与实验。

Windows 则不同。它拥有更广泛的生态覆盖、更丰富的兼容对象，也承载着大量真实工作流，但系统磁盘语义更加严格。要让一套能力真正进入 Windows 的磁盘体系，往往不能停留在普通应用层，而必须经过专门的驱动层、设备模型与更正式的挂载链路。

YumeDisk 的意义，就在于正面解决这个问题。它不是绕开 Windows 的磁盘语义要求，而是在 Windows 生态下把这层壁垒真正打通。

当这层壁垒被打通之后，最终呈现给上层的，不只是一个驱动或一个工具，而是系统磁盘操作可以被稳定地暴露到应用层。由此，应用不必重新发明一套存储工作流，而是可以围绕统一的磁盘语义，承接更广泛的介质能力。

之所以要把这条通道交给应用层，是因为很多真正的使用场景并不缺“存储位置”，而是缺“系统级可用性”。只有当应用能够站在标准磁盘语义之上组织自己的数据、状态与流程时，存储能力才不再是零散的外挂组件，而能够自然进入现有软件、现有工具链与现有工作流。这样一来，介质的变化就可以尽量留在底层，而应用层获得的仍然是统一、稳定且可持续扩展的磁盘操作入口。

## 项目特点

- 以磁盘语义为中心，而不是以文件传输语义为中心
- 强调统一的挂载和管理体验
- 允许不同介质能力在同一体系中被承载
- 适合逐步演进，而不是一次性押注单一形态

## 当前能力

从当前项目状态看，YumeDisk 已经不是停留在概念层的设计草案，而是在沿着可运行、可管理、可扩展的方向持续收敛。

它已经具备清晰的本地主线，也为更进一步的远端块设备能力打下了基础。当前已经覆盖的典型介质方向包括内存、文件与网络，它们分别对应临时介质、本地持久介质与远端能力入口，也构成了项目继续向外扩展的起点。

## 项目的潜力

YumeDisk 的价值不只在于“再提供一种存储方式”，更在于它为磁盘能力提供了一个可以继续生长的承载面。

这意味着它后续可以朝更多方向自然延伸，例如更丰富的介质类型、更完整的远端能力、更统一的客户端体验，以及更适合开发、测试、隔离与正式部署的使用方式。

它最重要的特点，在于具备继续成长为平台的可能，而不是停留在一个孤立工具或一次性实验项目。对 Windows 而言，这种价值尤为明显，因为一旦磁盘语义壁垒被打通，后续许多存储能力就有了进入系统工作流的统一入口。

除了目前已经覆盖的常见介质能力之外，其他介质形态同样值得继续探索。YumeDisk 也欢迎更多开发者一起参与，将更多存储想象力带入统一的磁盘语义体系之中。

## 总结

如果只用一句话概括，YumeDisk 想做的是：

在 Windows 生态下打通原本难以跨越的磁盘语义壁垒，将系统磁盘操作真正交到应用层手中，并把存储能力组织成可用、可管理、可扩展的磁盘系统体验。
